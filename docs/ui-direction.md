# UI Direction

## Design Goal

The app should feel:

- premium
- local
- tactile
- map-first
- mobile-native even though it is a web app

It should not feel like a generic SaaS dashboard or default map wrapper.

## Core Split

### Ride Mode

Visual mood:

- crisp
- bright
- functional
- high-contrast

Behavior:

- one main route
- one backup route
- big top turn card
- fast access to `Report`, `Reroute`, `Pause`

### Explore Mode

Visual mood:

- atlas-like
- warm
- collectible
- layered

Behavior:

- richer filters
- stronger cards
- thematic route discovery
- community activity and progress

## Inspiration Direction

Use inspiration, not imitation.

Good influences:

- Waze-style shared-city energy
- curated bicycle editorial sites
- subtle game-map atmosphere
- card systems that make places feel collectible

From the provided references, the best takeaway is:

- bold hero composition
- clean card hierarchy
- premium spacing
- strong mood without losing usability

## Typography Direction

- avoid default system-feeling combinations
- use a more expressive heading face for `Explore Mode`
- keep a highly legible body font for all utility text
- make sure Cyrillic and Latin both render well

## Color Direction

Suggested split:

- `Ride Mode`: strong daylight colors, clear route lines, alert chips
- `Explore Mode`: warmer paper tones, dark ink, muted accents, selected bright markers

Do not overdo sepia in navigation screens.

## Motion Direction

- subtle map and sheet transitions
- route reveal animation
- badge and achievement moments after the ride, not during
- no distracting motion while navigation is active

## Themed Route Cards

Themed cards should feel special and local.

Examples:

- `Historical route`
- `Beer + snack`
- `Шкембе route`
- `Coffee + rack`
- `Shade route`
- `Street art route`

Each card should show:

- why it is interesting
- safety level
- elevation
- surface notes
- estimated time

## UX Guardrails

- never force long forms on mobile
- prefer chips, toggles, and short notes
- use bottom sheets for map actions
- keep all critical actions reachable with one hand
- separate fun discoveries from hazard alerts

## Success Check

The UI is working if:

- it feels stylish in screenshots
- it feels calm while riding
- it still works in BG / EN / RU
- it is memorable enough that people want to show it to friends
